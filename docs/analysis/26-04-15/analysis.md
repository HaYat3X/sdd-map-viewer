## 潜在要求の仮説リスト

### 注：以下は仮説です。確定要件ではありません。

#### 1. **データプレビュー・属性確認機能**（R5 関連）
- **仮説内容**: インポート前に属性情報をテーブル形式で表示し、ユーザーが確認できる機能
- **仮説根拠**: 「インポート前に属性情報を確認したいみたいです」という発言から推定
- **次回確認項目**: 
  - 属性情報の表示形式（テーブル、JSON、プレビュー画面）
  - 属性情報に対する操作（フィルタリング、ソート、マッピング）

#### 2. **大規模データ対応・パフォーマンス最適化**（R3 関連）
- **仮説内容**: ファイルサイズ制限なしのため、MB～GB 規模のファイルを扱う際に非同期処理・ストリーミング処理が必要な可能性
- **仮説根拠**: 「大きいファイルも想定して欲しい」という発言
- **次回確認項目**: 
  - 実際に想定するファイルサイズの目安（MB か GB か）
  - Feature 数の規模（数百か数千か）
  - 処理時間の目安（即座か数秒か）

#### 3. **座標系変換・CRS 対応**（R8 関連）
- **仮説内容**: 複数ファイルが異なる座標系で入力される可能性。マージ時に WGS84 に統一するには、入力ファイルの座標系を自動検出・変換するロジックが必要
- **仮説根拠**: 「GeoJSONの座標系が違う可能性がある。メインの座標系は緯度経度に統一する」
- **次回確認項目**: 
  - 入力ファイルで想定される座標系のバリエーション（Web Mercator？ UTM？）
  - 座標系を自動検出するのか、ユーザーに指定させるのか

#### 4. **重複排除ロジックの詳細**（R7 関連）
- **仮説内容**: 複数ファイルをマージするとき、重複 Feature の判定基準（座標か プロパティ ID か）を決定する必要
- **仮説根拠**: 「どっちでもいい。判断は任せる」という発言
- **次回確認項目**: 
  - 重複判定の基準（完全一致か、近似座標か、プロパティ ID か）
  - 重複時の処理（削除か、マージか、警告表示か）

#### 5. **Feature クリック時の動作詳細**（R17 関連）
- **仮説内容**: 地図上の Feature をクリックしたときに、属性情報をポップアップやサイドバーで表示する機能
- **仮説根拠**: 「Feature をクリックしたときの動作は？（情報表示？）」という質問に「全部必要」と回答
- **次回確認項目**: 
  - 情報表示の形式（ポップアップか、サイドパネルか、テーブル行選択か）
  - 表示する属性の範囲（全属性か、特定属性のみか）

#### 6. **初期表示位置の決定ロジック**（R18 関連）
- **仮説内容**: マップ初期表示時の中心座標・ズームレベルを、マージ結果の Bounding Box から自動計算するのか、ユーザーが設定するのか
- **仮説根拠**: 「初期表示位置（中心座標・ズームレベル）のデフォルト値は？」という質問
- **次回確認項目**: 
  - 自動計算する場合：Bounding Box から中心・ズーム値の算出方法
  - ユーザー設定する場合：UI/UX（どこで設定するのか）

#### 7. **UI フレームワークの選定**（R19, R20 関連）
- **仮説内容**: Vue.js + UI フレームワーク（具体的な名称は未決定）+ TypeScript で実装
- **仮説根拠**: Charter で「Vue.js（プレーン）+ UIフレームワーク + ライブラリ」と記載。Hearing で「お任せ」と回答
- **次回確認項目**: 
  - UI フレームワークの具体的選択（Bootstrap か Vuetify か TailwindCSS か）
  - ダークテーマ・ライトテーマ対応の要否
